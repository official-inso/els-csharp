# Contributing

Thanks for your interest in improving the C# SDK for ELS.

## Development setup

Requirements:

- .NET SDK 8.0 or later (multi-target build covers `netstandard2.0`, `netstandard2.1`, `net6.0`, `net8.0`, `net9.0`).
- A POSIX shell or PowerShell to run helper scripts.

```bash
dotnet restore
dotnet build -c Release
dotnet test -c Release
```

## Project layout

```
src/Inso.Els/                       — core SDK
src/Inso.Els.AspNetCore/            — ASP.NET Core middleware + DI extensions
src/Inso.Els.Extensions.Logging/    — ILogger provider
tests/                              — xUnit test projects
examples/                           — runnable samples (EN + RU)
docs/                               — field reference and architecture notes
scripts/                            — helper shell scripts
```

## Commit and PR guidelines

- Keep PRs focused. One concern per PR.
- Write commit messages in imperative mood (`Add disk buffer flush`, `Fix 429 retry`).
- Add tests for any behavior change.
- Run `dotnet test` and the project guard script (`scripts/check-no-ai-mentions.sh`) locally before opening a PR.

## House rules

This repository is a clean, vendor-neutral SDK. Please **do not** include any of
the following in source files, documentation, commit messages, PR descriptions,
or generated artifacts:

- References to AI assistants (Claude, Anthropic, ChatGPT, OpenAI, Copilot, etc.).
- `Co-Authored-By:` trailers naming AI tools.
- "Generated by ...", "Made with AI", or similar attribution.

A CI workflow (`.github/workflows/no-ai-mentions.yml`) enforces this on every
push and pull request. The same check is available locally:

```bash
bash scripts/check-no-ai-mentions.sh
```

If you have questions, open an issue.
